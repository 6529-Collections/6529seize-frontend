# Network nerd tdh view toggle

PR: https://github.com/6529-Collections/6529seize-frontend/pull/2697

Branch: `network-nerd-tdh-view-toggle`
Target: `main`

## Task Summary

Add and refine the Network Nerd consolidated TDH view, including boosted vs
unboosted TDH support and follow-up UX fixes for the staging review.

## Frontend Change Summary

- Routes/views/components changed:
  - `/network/nerd` client route now uses the same `NetworkPageLayout` shell as
    `/network` so margins and page padding match.
  - Network Nerd header copy now reads `Network Nerd` and uses the compact
    `/network` heading scale instead of a large hero heading.
  - Leaderboard stat-card data is cached across focus-route remounts so the
    top-right TDH cards do not drop back to loaders when switching between
    Cards Collected and Interactions.
  - Header search now exposes one `Network Nerd` page result while retaining
    cards/interactions search aliases for that single entry.
- Generated models/API client changes: none in this follow-up.
- User-visible behavior changes:
  - Cards Collected and Interactions switches update the table view without
    visually refreshing the top TDH stat cards.
  - Search no longer shows separate Network Nerd Cards Collected and Network
    Nerd Interactions entries.
