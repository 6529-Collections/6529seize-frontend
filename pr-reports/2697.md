# Network nerd tdh view toggle

PR: https://github.com/6529-Collections/6529seize-frontend/pull/2697

Branch: `network-nerd-tdh-view-toggle`
Target: `main`

## Task Summary

Add and refine the Network Nerd consolidated TDH view, including boosted vs
unboosted TDH support and follow-up UX fixes for the staging review.

## Frontend Change Summary

- Routes/views/components changed:
  - `/network/nerd` client route now uses the same `NetworkPageLayout` shell as
    `/network` so margins and page padding match.
  - Network Nerd header copy now reads `Network Nerd` and uses the compact
    `/network` heading scale instead of a large hero heading.
  - Cards Collected and Interactions switches use the supported Next.js native
    history API path instead of a Next route transition, so the page/stat area
    stays mounted while the URL still updates.
  - Header search now exposes one `Network Nerd` page result while retaining
    cards/interactions search aliases for that single entry.
  - Cards Collected and Interactions leaderboard tables now use native table
    markup with Tailwind classes for table color, spacing, alternating rows, and
    header weight instead of `react-bootstrap` table markup and broad
    CSS-module table overrides.
- Generated models/API client changes: none in this follow-up.
- User-visible behavior changes:
  - Cards Collected and Interactions switches update the table view without
    visually refreshing the top TDH stat cards.
  - Search no longer shows separate Network Nerd Cards Collected and Network
    Nerd Interactions entries.
  - Leaderboard header and numeric cells remain readable in both Cards
    Collected and Interactions modes.
  - Leaderboard table headers use Tailwind `font-semibold`/600 weight instead
    of the previous heavier `font-weight: 800` CSS override.

## Release Notes

- Network Nerd now visually aligns with the main Network page.
- Network Nerd focus switches no longer refresh the top stat cards.
- Global search now shows one Network Nerd page entry instead of deep-link
  duplicates.
- Network Nerd leaderboard tables preserve the original dark table contrast in
  both Cards Collected and Interactions modes.
- Cards Collected now uses the same narrower collector-column behavior as
  Interactions and no longer over-allocates table width beyond its container.
- Network Nerd leaderboard table headers now use a lighter 600 weight and the
  table rendering no longer depends on `react-bootstrap` table markup or the
  previous broad CSS-module table override.
- No API, environment, generated-model, or deployment compatibility changes.

## Residual Risks And Follow-ups

- Local frontend dev-server visual inspection was not run because the refreshed
  staging workflow explicitly says not to start a local dev server for
  staging-skill work.
- Unauthenticated staging smoke can only verify route availability and access
  gating; authenticated browser verification covered the table contrast and
  header-weight follow-up on the deployed staging build.
- The module-level stat cache suggested in the first follow-up was removed after
  review-bot feedback. Focus changes now avoid remounting via native history
  URL updates instead.
