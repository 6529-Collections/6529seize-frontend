# PR Report: The Memes Year Filters

PR: https://github.com/6529-Collections/6529seize-frontend/pull/2973
Branch: codex/the-memes-year-filters -> main
Related PRs: None

## Summary

Adds Year filtering to The Memes browse page, scopes the Season dropdown by the selected Year, and links the Year breadcrumb on meme detail pages back to the Year-filtered browse view.

## What Changed

- Added a Year dropdown to `/the-memes` with `All Years` plus the years derived from loaded Meme season data, including Year 0.
- Scoped the Season dropdown so all years show every season, while a selected year shows only that year's seasons with an `All Year N Seasons` action.
- Added `year` query-param support alongside the existing `szn` param and reset impossible year/season pairs back to the default filter state.
- Reused the existing backend multi-season `season` query behavior to fetch all cards in a selected Year.
- Refactored the old season-only grid dropdown into a reusable `FilterGridDropdown` plus a `MemeSeasonGridDropdown` wrapper.
- Made the detail-page Year breadcrumb link to `/the-memes` with the matching `year` filter while preserving locale-aware links.
- Updated The Memes browse documentation and help-index metadata for Year filtering.

## Frontend Impact

- Routes/views: `/the-memes` and `/the-memes/[id]`.
- Components: The Memes browse filters, Meme Calendar breadcrumb periods, reusable grid dropdown controls, and user-collected season filter wrapper.
- Generated API/client: Not affected.
- User-visible behavior: Users can browse all cards by Year, Season choices stay scoped to the active Year instead of showing disabled unrelated seasons, invalid URL filter combinations are cleaned up, and detail pages expose a clickable Year breadcrumb.

## Config / Env

No config or environment changes.

## Release Notes

The Memes browse page now supports Year filtering with Year-scoped Season choices, and meme detail Year breadcrumbs link back to the matching Year browse view.
