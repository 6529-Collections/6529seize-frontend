# PR Report: Native Profile Overlay Navigation

PR: https://github.com/6529-Collections/6529seize-frontend/pull/2794
Branch: codex/native-screen-stack-swipe -> main
Related PRs: None
Mode: staging-deploy

## Summary
Adds a scoped native-app overlay navigation path so opening profile routes from `/waves` or `/messages` preserves the underlying stream state and lets the user close the profile with an edge swipe.

## What Changed
- Added App Router `@nativeOverlay` parallel slots under `/waves` and `/messages`.
- Added intercepted profile route shims for profile root and catch-all profile subroutes.
- Added a private profile overlay delegator that reuses the existing profile root, tab, legacy redirect, and CMS route modules.
- Added a native overlay client component with edge-swipe close behavior and one-deep internal overlay navigation using `router.replace`.
- Documented the extension pattern for future preserved hosts.

## Frontend Impact
- Routes/views: native app navigations from `/waves` and `/messages` to `/:user` and `/:user/*`.
- Components: `NativeRouteOverlay` plus the new scoped App Router route slots.
- Generated API/client: Not affected.
- User-visible behavior: in the native app, profile links opened from waves/messages render as a full-screen overlay on top of the preserved stream; Back or an inward edge swipe returns to the original stream position.

None.

## Release Notes
Native app users can open profiles from waves or messages without losing their place, then close the profile with an edge swipe.
