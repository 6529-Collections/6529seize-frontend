# PR Report: Modernize public NextGen pages

PR: https://github.com/6529-Collections/6529seize-frontend/pull/3357
Branch: `codex/nextgen-tailwind-refresh` -> `main`
Related PRs: None

## Summary

Modernizes the public NextGen landing, collection, mint, token, rarity, display,
and provenance experiences around a shared responsive Tailwind visual language
while preserving the existing collection, token, marketplace, mint, and
generator behavior.

## What Changed

- Standardized public NextGen page frames, navigation, spacing, typography,
  controls, responsive stacking, loading states, empty states, and errors.
- Refreshed collection, artist, trait-set, art, mint, distribution-plan, token,
  download, viewer, rarity, and on-chain fallback surfaces.
- Migrated token and collection provenance to Tailwind-only activity rows that
  share the `LatestActivity` presentation and transaction controls, keep a
  meaningful event summary visible, and reserve disclosure for supplemental
  transaction detail.
- Rebalanced the collection overview card with denser copy, clearer metadata,
  a quieter distribution-plan action, and a softer desktop divider.
- Removed the remaining NextGen CSS modules and migrated the manager shell and
  residual admin controls to Tailwind utilities.
- Corrected collection Back/Forward tab routing, mint read synchronization,
  stale artist biographies, metadata image validation, owner loading, and
  request failure handling.
- Removed the homepage mission statement paragraph's default top margin.
- Expanded focused coverage and NextGen user/help documentation for the updated
  behavior.

## Frontend Impact

- Routes/views: public `/nextgen`, collection subroutes, mint and distribution
  plan, and token About, Provenance, Display Center, and Rarity views.
- Components: shared NextGen page frames, collection/token panels, activity
  rows, loading/error states, media controls, and navigation.
- Generated API/client: Not affected.
- User-visible behavior: modern responsive styling, clearer state feedback,
  accessible loading labels, balanced collection overview hierarchy, and
  provenance rows without placeholder separators or empty disclosures.

## Config / Env

No changes.

## Release Notes

Public NextGen pages now use a consistent responsive visual system. Collection
and token provenance share the compact activity-row design, always show the
event description before expansion, and failed data requests provide distinct
recovery states instead of appearing as empty data.

## Risks / Follow-Ups

The change spans most public NextGen views, so the primary residual risk is
visual regression across unusual collection data and narrow viewport sizes.
