# PR Report: Format Shared Pagination Page Counts

PR: https://github.com/6529-Collections/6529seize-frontend/pull/3293
Branch: `agent/format-pagination-page-counts` -> `main`
Related PRs: None

## Summary

Shared table pagination now displays locale-formatted current and total page counts, including on the Network page.

## What Changed

- Routed both values in the shared `Page X of Y` label through the existing canonical integer formatter.
- Added regression coverage for thousands separators on both values.

## Frontend Impact

- Routes/views: Network, community members, profile activity, collected items, and user activity views that use the shared paginator.
- Components: `CommonTablePagination`.
- Generated API/client: Not affected.
- User-visible behavior: Large page counts display with thousands separators, such as `Page 12,345 of 20,598`.

## Release Notes

Large current-page and total-page values in shared table pagination are now easier to read with thousands separators.
