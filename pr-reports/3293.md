# PR Report: Format Shared Pagination Page Counts

PR: https://github.com/6529-Collections/6529seize-frontend/pull/3293
Branch: `agent/format-pagination-page-counts` -> `main`
Related PRs: None

## Summary

Both shared pagination controls now display localized copy and locale-formatted current and total page counts, including on the Network and Network Nerd pages.

## What Changed

- Routed both values in the shared pagination label through the existing integer formatter using the active browser locale.
- Added localized pagination labels for all supported locales.
- Formatted the editable current-page field when it is not focused while preserving ungrouped numeric entry during editing.
- Centralized locale message construction so pagination keys are defined once rather than duplicated across dictionaries.
- Added regression coverage for translated copy, word order, and locale-specific thousands separators.
- Kept both editable pagination controls available if a localized template is malformed or omits a placeholder.

## Frontend Impact

- Routes/views: Network, Network Nerd, community members, profile activity, collected items, user activity, subscriptions report, The Memes, NextGen, Rememes, Meme Lab, downloads, distributions, and other views using either shared paginator.
- Components: `CommonTablePagination` and `Pagination`.
- Generated API/client: Not affected.
- User-visible behavior: Pagination labels and number grouping follow the active supported browser locale, such as `Page 12,345 of 20,598` in English and `Seite 12.345 von 20.598` in German.

## Release Notes

Shared pagination now uses localized labels and locale-appropriate thousands separators for large current-page and total-page values, including editable pagination on Network Nerd.
