# PR Report: Meme Calendar Position Fields

PR: https://github.com/6529-Collections/6529seize-frontend/pull/2978
Branch: codex/meme-calendar-position-fields -> main
Related PRs: None

## Summary

Adds per-division ordinal positions to public Meme Calendar API responses, using the existing timeline ranges so the calendar hierarchy logic remains unchanged.

## What Changed

- Added `position_in_season`, `position_in_year`, `position_in_epoch`, `position_in_period`, `position_in_era`, and `position_in_eon` to meme calendar mint responses.
- Derived each position from the existing resolved timeline range for that zoom level instead of changing core calendar range logic.
- Added an explicit SZN1 historical range guard so Cards 1-47 count from mint 1 across the Year/Epoch/Period/Era/Eon 0 ranges.
- Covered the new response shape plus exact Card 516 and SZN1/Card 47 positions in API route tests.
- Updated the minting calendar docs and help index to include the new response fields.

## Frontend Impact

- Routes/views: `/api/meme-calendar/next`, `/api/meme-calendar/current`, and `/api/meme-calendar/{id}` responses.
- Components: Not affected.
- Generated API/client: Not affected.
- User-visible behavior: API consumers can show a card's ordinal position within its SZN, Year, Epoch, Period, Era, and Eon.

## Config / Env

None.

## Release Notes

The Meme Calendar API now returns ordinal position fields for each calendar division, including historical SZN1 cards and structured timeline cards such as Card 516.
