# PR Report: Migrate WordPress pages from legacy styling

PR: https://github.com/6529-Collections/6529seize-frontend/pull/3185
Branch: `codex/wp-legacy-article-pilot` -> `main`
Related PRs: None

## Summary

This PR pilots and expands the WordPress migration cleanup strategy by replacing selected scraped Avada/Fusion pages with native 6529 rendering and route-local typed content data.

## What Changed

- Added reusable migrated WordPress article and static page renderers with `data-content-source="migrated-wordpress"` markers for easy future audits.
- Moved selected blog, news, education, and about page copy, metadata, images, videos, captions, and key links into typed migrated-WordPress content data.
- Removed those routes' dependency on WordPress/Fusion head assets, inline styling, duplicated metadata blocks, and the legacy WordPress footer.
- Let the migrated shells use the available app content width while preserving readable max widths for text and media columns.
- Added focused renderer coverage for article metadata, static pages, media, captions, quotes, rich HTML blocks, and the migrated-source marker.

## Config / Env

No config or environment changes.

## Release Notes

Selected legacy WordPress pages now render through native 6529 layouts while keeping migrated content available at the same URLs.

## Risks / Follow-Ups

This migration intentionally does not preserve WordPress visual parity. It preserves content and URLs, leaves migrated pages auditable through the shared components/source marker, and leaves untouched WP-migrated routes for later batches.
