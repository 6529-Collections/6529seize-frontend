# PR Report: Android Mobile Chat And QR Scanner Fixes

PR: https://github.com/6529-Collections/6529seize-frontend/pull/2885
Branch: codex/android-inner-chat-qr-fixes -> main
Related PRs: None

## Summary
Fixes Android mobile detail-view spacing in wave/message chats, improves Android in-app QR scanner detection, and makes intentional scanner cancellation silent instead of showing an error toast.

## What Changed
- Removed the Android-only fixed chat height reserve from the shared inner chat layout.
- Configured Android QR scans to use native ML Kit scanning with the back camera and adaptive orientation while preserving app-level URL/deep-link validation.
- Suppressed the native scanner cancellation error when users close the scanner intentionally.
- Added focused coverage for the Android layout regression, Android scanner options, and scanner cancellation behavior.

## Frontend Impact
- Routes/views: Android mobile wave and message detail views; in-app QR scanner entry points.
- Components: `LayoutContext`, `HeaderQRScanner`.
- Generated API/client: Not affected.
- User-visible behavior: Android detail chats no longer keep an oversized bottom gap, Android QR scanning should resolve valid codes more reliably, and closing the scanner no longer shows an error toast.

## Config / Env
None.

## Release Notes
Android mobile users should see tighter wave/message detail layouts, more reliable in-app QR scanning, and a quieter scanner cancel flow. iPhone QR scan behavior remains QR-focused, with the cancellation toast suppressed for the same intentional close action.
