# PR Report: Dedicated 6529 Apps page

- PR: https://github.com/6529-Collections/6529seize-frontend/pull/3244
- Branch: `codex/6529-apps-page` → `main`
- Related PRs: None

## Summary

Adds a discoverable home for the 6529 Mobile and Desktop apps and removes app downloads from the Share modal.

## What Changed

- Added `/about/6529-apps` with separate, responsive Mobile and Desktop sections.
- Added 6529 Apps to the About navigation.
- Added search aliases for 6529 Apps, 6529 Mobile, 6529 Desktop, apps, mobile, and desktop.
- Moved reusable download behavior out of Share-owned components.
- Reduced Share to Connection and Current URL sharing.
- Centered and stacked the Mobile store badges without nested card chrome, and clarified Desktop links with external-link icons.
- Hardened desktop manifest handling with cancellation, timeouts, version validation, partial-failure diagnostics, and controlled query retries.
- Completed message-key typing and localized the touched `/open-mobile` shell copy.
- Added route, navigation, search, download, and Share-modal coverage.

## Frontend Impact

- New user-visible route: `/about/6529-apps`.
- About navigation and global page search now expose the apps destination.
- Existing mobile deep-linking and desktop release-manifest sources are preserved.
- Malformed or unavailable platform manifests are omitted without hiding valid platform downloads.
- No generated API client changes.

None.
