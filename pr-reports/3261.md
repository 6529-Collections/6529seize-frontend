# PR Report: Link Main Stage winners to Meme cards

PR: https://github.com/6529-Collections/6529seize-frontend/pull/3261
Branch: `codex/main-stage-meme-links` -> `main`
Related backend PR: https://github.com/6529-Collections/6529seize-backend/pull/1749

## Summary

Winning submissions in The Memes Main Stage now show which Meme card they produced and link directly to that card from both the open drop and the wave Winners tab. Mapped Meme card pages also link back to their originating Main Stage submission.

## What Changed

- Preserve `submission_context.meme_card_id` on a frontend-owned hydrated V2 drop view instead of adding it to the legacy winning-context contract.
- Show a compact `The Memes #N` pill on mapped winners, including beside the
  `Minted on The Memes` outcome on the dedicated drop view.
- Stop the Winners-card parent click so the Meme link navigates directly to the card.
- Omit the link when the backend has no known relationship.
- Resolve a Meme card's explicit mapping through the paired backend endpoint
  and show a `Main Stage submission` section between Arweave links and
  Properties under Additional Details.
- Build the return URL with the runtime `memes_wave_id` setting, so no frontend wave ID or environment variable is introduced.
- Keep the frontend storage-agnostic: it consumes only the optional V2 mapping and never infers a Meme ID from winner order.
- Refresh the frontend OpenAPI contract and generated models from the paired backend branch.
- Document the Main Stage-specific behavior in the Winners help content.
- Center single-drop author activity badges with the avatar/name/level row and
  optically align Additional Details section icons with their headings.

## User Impact

Users can move directly from a Main Stage winner to its minted Meme card without searching the collection manually. Other waves are unchanged.
