# PR Report: Subscriptions Report Active Drop Links

PR: https://github.com/6529-Collections/6529seize-frontend/pull/3142
Branch: codex/subscriptions-report-active-labels -> main
Related PRs: None

## Summary
Subscriptions Report now uses clearer active-drop terminology, opens Meme Card
routes from native grid rows, and gives unresolved upcoming Meme Card pages the
same subscription awareness widget used on home/current card surfaces. The
report and About Subscriptions page now also use mobile-specific layouts instead
of squeezing desktop controls into narrow screens.

## What Changed
- Renamed Active Drop count columns from `Projected` / `Actual` to
  `Subscribed` / `Airdropped`.
- Increased spacing between the Active Drop table and the Upcoming Drops
  heading.
- Made Active, Upcoming, and Past subscriptions-report rows a single native
  link to `/the-memes/{token_id}` using grid rows instead of table/link
  overlays.
- Added subscription awareness above the next-mint calendar for unresolved
  upcoming `/the-memes/{id}` routes.
- Reworked report rows on mobile so Active Drop keeps labeled stats while
  Upcoming and Past rows show compact right-aligned counts without repeating
  "Subscriptions" on every row.
- Added the `x` count prefix across report count values, e.g. `x559`, `x17`,
  and `x558`.
- Changed Active Drop detection to use `/api/meme-calendar/current` live mint
  status instead of comparing redeemed `mint_date` with the current day, so
  live drops that cross midnight stay in Active Drop until `mint_end`.
- Moved the About Subscriptions subscribe/manage action into the page content
  below the title so the About dropdown keeps the same standalone top-right
  placement as other About pages.
- Updated route docs and help records for the renamed counts, row links, and
  upcoming-card subscription widget.

## Frontend Impact
- Routes/views: `/tools/subscriptions-report`, `/the-memes/{id}` unresolved
  upcoming-card fallback, `/about/subscriptions`.
- Components: `components/subscriptions-report/SubscriptionsReport.tsx`,
  `components/subscriptions-report/SubscriptionsReportRows.tsx`,
  `components/the-memes/UpcomingMemePage.tsx`,
  `components/about/About.tsx`,
  `components/about/AboutSubscriptions.tsx`,
  `components/about/AboutContentsDropdown.tsx`.
- Generated API/client: Not affected.
- User-visible behavior: Active Drop shows subscribed versus airdropped counts;
  report rows are navigable; upcoming card URLs show subscription awareness.

## Config / Env
None.

## Release Notes
Subscriptions Report now labels Active Drop counts as Subscribed and Airdropped,
adds more breathing room before Upcoming Drops, and lets users open Meme Card
routes from native report rows. Upcoming Meme Card URLs also show the subscription
awareness widget above the mint calendar. Mobile layouts now keep report counts
compact, show count values with an `x` prefix, and move the About Subscriptions
action out of the sticky dropdown area on narrow screens. Active Drop now
follows the live meme-calendar mint window, so a card remains active until its
calendar `mint_end`.
