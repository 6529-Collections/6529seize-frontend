## Task Summary

Make REP categories discoverable as a first-class page, let users search/select a category, and show profile and Wave REP analytics without forcing a second wave search.

## Frontend Changes

- Adds `/rep/categories` and `/rep/categories/[category]`.
- Adds REP Categories to Network navigation below Wave Score and to header search page discovery.
- Changes the category explorer to search first, show results, and scroll/focus the REP Category detail after a result is selected.
- Adds suggested active categories to the empty landing state.
- Renames the detail heading from Global REP Category to REP Category and adds a back-to-search link on deep links.
- Replaces Wave REP wave search with category-wide Wave REP analytics: waves using the category and contributors for those waves.
- Shows wave PFPs and profile avatars in Wave REP analytics.
- Adds existing profile hover cards to REP category user links.
- Removes the ambiguous table status label, aligns scalar table columns consistently, and removes unlabeled contributor contribution chips from the leading contributors column.
- Adds infinite scrolling to paginated profile and wave lists, and labels preview sections as previews.
- Sorts REP rankings by absolute REP impact, with signed value used as a tie-breaker.
- Updates generated frontend API models/client from the backend OpenAPI contract.
