# PR Report: Auth Upgrade Menu and Tech Navigation

Related PRs: None
Mode: staging-deploy

## Summary

This frontend-only change makes the wallet-authentication upgrade easier to reach during the silent rollout and improves discovery of the Tech pages that explain the auth update.

## What Changed

- Legacy session-v1 users now keep a manual "Upgrade Authentication" profile-menu action even when the backend migration deadline is unset.
- Disconnected legacy web sessions can open the upgrade prompt from the profile menu and reconnect the wallet before signing.
- The automatic upgrade modal remains gated by backend rollout settings, so silent release behavior is preserved.
- The About navigation now includes Tech under Resources, which also makes `/about/tech` available in header search.
- The Active Technical Notes wallet-authentication card on `/about/tech` is fully clickable.

## Frontend Impact

- Routes/views: `/about/tech` card behavior and About navigation discoverability.
- Components: `Auth`, profile menu, header search page catalog through sidebar sections.
- Generated API/client: Not affected.
- User-visible behavior: Users who dismissed or did not receive the automatic upgrade prompt can manually start auth upgrade from the profile menu when they are on a legacy session.

## Config / Env

No new or changed config.

## Release Notes

Users on older wallet-auth sessions get a clearer manual upgrade path from the profile menu. The Tech page is easier to find from About navigation and search, and its wallet-authentication note behaves like a full clickable card.
