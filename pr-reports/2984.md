# PR Report: Home Mint Subscription Awareness

PR: https://github.com/6529-Collections/6529seize-frontend/pull/2984
Branch: codex/home-subscription-awareness -> main
Related PRs: None

## Summary

Home and current The Memes detail pages now surface The Memes subscription affordance more broadly so connected users can see read-only subscription status and profile actions even when they are not already subscribed, while disconnected or proxy-context users see awareness and setup paths without embedding the profile editor.

## What Changed

- Latest Drop passes the displayed Meme Card id into a read-only subscription awareness widget, avoiding the upcoming-meme status endpoint for current or already-dropped cards.
- Next Drop keeps the canonical next mint behavior and shows the same read-only subscription awareness widget in the top panel.
- The current/latest `/the-memes/{id}` detail page shows the same awareness-only subscription row below the mint countdown.
- Awareness-only rows use a blue `Subscription Minting` box with a read-only toggle, optional `xN` user subscribed count, `xN subscribers` total, `Set up`/`Manage` profile-subscriptions action, and the question-mark info link.
- The awareness toggle is a non-editable status indicator with shared app tooltip styling, and subscriber-count loading uses the existing `DotLoader` without shifting the row layout.
- The home/latest-card widget no longer embeds the profile subscription editor, balance label, eligibility selector, quantity selector, or mutation controls.
- `/about/subscriptions` shows connected users a primary `Manage` action on the left of the about-section nav row, and shows disconnected visitors a primary `Connect to Subscribe` action that forwards them to profile subscriptions after wallet connection and signature authentication.
- The profile subscriptions tab keeps its `Learn More` link visible as a right-aligned secondary action in the `Subscribe` header row.
- Home docs now mention the expanded subscription affordance.

## Frontend Impact

- Routes/views: `/` home Latest Drop and Next Drop mint panels, current `/the-memes/{id}` detail pages, and `/about/subscriptions`.
- Components: home mint subscription widget, compact Meme subscription row, Meme detail page mint panel, about contents navigation, and profile subscriptions header.
- Generated API/client: Not affected.
- User-visible behavior: subscription awareness is shown to more visitors, connected users can reach `/{user}/subscriptions` from the home mint panel, and disconnected users are routed there only after completing wallet signature authentication.

## Config / Env

No config or environment changes.

## Release Notes

The home mint panel and current The Memes detail page now make The Memes subscriptions more discoverable for connected, unsubscribed, disconnected, and proxy-context visitors, while preserving the existing platform/country hide rule and subscription action disablement rules. Profile subscription setup links now wait for authentication before navigating to the subscriptions tab.
