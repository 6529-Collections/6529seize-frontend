# PR Report: Silence Bootstrap Sass Dependency Warnings

Related PRs: None
Mode: pr-ready

## Summary

Restores the Bootstrap Sass package import so Bootstrap is loaded through the configured Sass load path and remains classified as a dependency for `quietDeps`. Adds a build guard so future reversions to a relative `node_modules` Bootstrap import fail before deployment logs can regress.

## What Changed

- Changed `styles/seize-bootstrap.scss` back to `@use "bootstrap/scss/bootstrap"`.
- Restored the inline rationale that this import works with `sassOptions.loadPaths` and `quietDeps`.
- Added `scripts/guard-bootstrap-sass-import.cjs` to enforce the Bootstrap import and the required Next Sass options.
- Wired `guard:bootstrap-sass` into `prebuild`, so `./bin/6529 run build` fails fast if the import/config contract is broken.
- Left the existing AGENTS.md Bootstrap Sass guidance unchanged.

## Frontend Impact

- Routes/views: Not directly affected.
- Components: Global Bootstrap styles compile through the existing Sass pipeline.
- Generated API/client: Not affected.
- User-visible behavior: No intended UI change; this is build-log and Sass dependency-classification cleanup.

## Config / Env

No new config or env variables. The change depends on the existing `sassOptions.loadPaths` and `quietDeps` configuration in `config/nextConfig.ts`, now guarded by `prebuild`.
