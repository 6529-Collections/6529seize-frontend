## Task Summary
Fix inconsistent hide/restore link-preview behavior in wave posts by sending the intended final visibility state and reconciling the optimistic UI with the server response.

## Frontend Change Summary
- Changed link-preview actions to call `onToggle(true)` for hide and `onToggle(false)` for restore.
- Optimistically writes the desired `hide_link_preview` state instead of flipping local state.
- Reconciles the optimistic store with the returned `ApiDrop.hide_link_preview`.
- Guards overlapping requests for the same drop across hook instances, with owner cleanup on unmount.
- Regenerated frontend API model for `ApiToggleHideLinkPreviewRequest`.

## Frontend Impact Summary
- Routes/views/components changed: wave post link-preview controls and action menus.
- Generated models/API client changes: new `ApiToggleHideLinkPreviewRequest`; OpenAPI updated for optional request body.
- User-visible behavior: hide and restore link-preview actions should immediately reflect the intended final state and stay consistent after websocket updates or refresh.
