# PR Report: Float Mobile Waves Quick Vote Above Dock

PR: https://github.com/6529-Collections/6529seize-frontend/pull/2880
Branch: codex/waves-uncast-floating-overlay -> main
Related PRs: None

## Summary

This refines the mobile `/waves` bottom layout so the waves list remains a continuous page surface behind the floating dock, while the Uncast Power quick-vote control floats above the dock instead of occupying an in-flow footer band.

## What Changed

- Mobile waves keeps a bounded flex shell while its scroll container remains the continuous visible route surface behind the dock.
- The waves scroll area switches between dock-only clearance and quick-vote overlay clearance based on whether the floating footer is actually available.
- `MemesWaveFooter` gained a mobile floating mode while preserving the existing desktop/sidebar footer and collapsed pill behavior.
- The floating footer bottom offset is applied through a shared inline style so the overlay does not depend on generated Tailwind arbitrary-value output for viewport placement.
- The floating footer now measures the actual mobile dock top edge and tracks it during resize and dock transition frames so the card sits closer to the expanded dock and moves down with the compact dock.
- The dock measurement rebinds if the mobile dock DOM node is replaced, and high-frequency transition tracking writes the measured bottom directly to the floating layer to keep the movement smoother.
- Dock replacement observation is scoped to the mobile bottom-nav root so live feed mutations do not schedule footer measurements.
- The global New Version toast now uses the same measured dock offset on mobile, keeping it just above the floating dock as the dock expands or compacts.
- Focused tests cover the bounded mobile waves scroll target contract, floating footer availability, measured dock movement, and floating quick-vote footer layer.

## Frontend Impact

- Routes/views: mobile `/waves`.
- Components: `BrainMobileWaves`, `MemesWaveFooter`.
- Generated API/client: Not affected.
- User-visible behavior: the waves list continues behind the dock, the Uncast Power box appears above the dock without a full-width black footer strip, tracks the dock height while scrolling, and extra scroll clearance is only applied while that box is available.

## Config / Env

No config or environment changes.
