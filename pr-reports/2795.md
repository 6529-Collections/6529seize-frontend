# PR Report: Increase connected profile limit to five

PR: https://github.com/6529-Collections/6529seize-frontend/pull/2795
Branch: codex/connected-accounts-limit-5 -> main
Related PRs: None
Mode: staging-deploy

## Summary

Raises the frontend connected-profile storage cap from three accounts to five accounts so users can keep more authenticated profiles available for switching.

## What Changed

- Updated the shared connected-profile limit constant from `3` to `5`.
- Updated auth utility coverage so five stored accounts are accepted and a sixth account is rejected.
- Updated wallet/account controls documentation to describe the five-profile limit and add-account visibility.

## Frontend Impact

- Routes/views: Account controls wherever connected profile switching and add-account actions are available.
- Components: Existing web and app account-control surfaces inherit the new cap through shared auth context state.
- Generated API/client: Not affected.
- User-visible behavior: Users can store up to five connected profiles before add-account controls are hidden and new accounts are rejected.

## Config / Env

No config or environment changes.

## Release Notes

Users can now keep up to five connected profiles available for account switching instead of three.
