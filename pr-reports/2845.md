# PR Report: Polish wave sidebar spacing and grouped message timestamps

PR: https://github.com/6529-Collections/6529seize-frontend/pull/2845
Branch: codex/wave-ui-polish -> main
Related PRs: None
Mode: staging-deploy

## Summary

Tightens wave sidebar spacing and adds timestamp access for grouped chat messages that do not render their own author row.

## What Changed

- Renamed the sidebar preview section to Worth Checking Out, tuned its header/avatar spacing, and distributed visible avatar previews across the strip.
- Rebuilt sidebar wave/subwave name and timestamp rows as a compact text stack, keeping score/actions outside that stack so timestamps sit directly under names without negative margins or subwave clipping.
- Added a muted grouped-message timestamp affordance: desktop hover/focus reveals it in the author-column space, and mobile left-swipe reveals it on the right.
- Changed reply/original-message jumps to scroll the drops container directly instead of calling viewport-level `scrollIntoView`.

## Frontend Impact

- Routes/views: Wave chat and web/app wave sidebars.
- Components: Wave drop rendering, serial-number jump scrolling, Worth Checking Out sidebar preview strip, and sidebar wave/subwave rows.
- Generated API/client: Not affected.
- User-visible behavior: Grouped chat messages can expose their sent time without adding a permanent author row; reply jumps should no longer disturb page-level layout around the composer.

## Config / Env

No config or environment changes.
