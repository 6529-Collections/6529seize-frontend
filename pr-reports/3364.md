# PR Report: Link preview stack refactor

PR: https://github.com/6529-Collections/6529seize-frontend/pull/3364
Branch: `agent/split-link-preview-stack` -> `main`
Related PRs: None

## Summary

The frontend link-preview stack is split into provider-owned modules while preserving its existing rendering behavior and improving provider-specific failure handling.

## What Changed

- Extracted oversized 6529, GitHub, Twitter/X, Farcaster, Google Workspace, marketplace, Pepe, and Wikimedia preview logic into focused modules.
- Kept preview parsing, media rendering, loading states, and provider cards independently testable.
- Corrected Wikimedia Commons file-page detection so `.jpg` page URLs render Commons cards instead of being treated as raw images.
- Corrected Compound v2 and v3 contract interfaces and the cETH registry checksum so supported market previews can load live on-chain data.

## Frontend Impact

- Routes/views: Wave and drop content that renders supported external links.
- Components: Provider-specific preview cards and markdown link routing.
- Generated API/client: No contract changes.
- User-visible behavior: Supported Commons and Compound links now render their intended previews.

## Config / Env

No new or changed configuration.

## Release Notes

Link previews are organized into smaller provider-specific modules. Wikimedia Commons file pages and the supported Compound market links now render working previews.
