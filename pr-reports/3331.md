# PR Report: Shared On-Chain Transaction Modal and Notifications Loading

PR: https://github.com/6529-Collections/6529seize-frontend/pull/3331
Branch: `codex/onchain-modal-notifications-loader` -> `main`
Related PRs: None

## Summary

This PR removes the false disconnected-wallet flash from fresh-tab notification loads and consolidates subscription top-up and Drop Forge transaction status presentation into one accessible shared modal.

## What Changed

- Shows a neutral centered loader while `/notifications` restores wallet auth, reserving the disconnected-wallet prompt for settled unauthenticated state.
- Adds a shared transaction modal for wallet confirmation, submitted, success, and error states.
- Centralizes portal rendering, terminal-state dismissal, body scroll locking, nested-modal-safe focus management, transaction links, and keyboard-accessible long-error containment.
- Migrates subscription top-ups and Drop Forge to the shared presentation without changing transaction submission or receipt monitoring.
- Keeps normalized subscription receipt failures in the shared error modal,
  preserves dismissal behavior across in-place transaction status changes, and
  formats touched card counts through the locale-aware integer helper.
- Documents active connect/reconnect loading behavior and the current `en-US`
  fallback debt for the new shared status surfaces.

## Frontend Impact

- Routes/views: `/notifications`, profile subscription top-ups, and Drop Forge launch actions.
- Components: new shared `OnchainTransactionModal`; existing subscriptions and Drop Forge consumers now provide state and consumer-specific copy.
- Generated API/client: Not affected.
- User-visible behavior: authenticated notification loads no longer show a transient connect prompt; both on-chain flows use the same accessible transaction dialog behavior.

## Config / Env

None.

## Release Notes

Fresh-tab notification loads now wait for wallet restoration before deciding whether to show a connect prompt. Subscription top-ups and Drop Forge actions share a consistent transaction-status modal with improved spacing, alignment, dismissal, focus handling, transaction links, and long-error readability.
