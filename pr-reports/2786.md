# PR 2786 Report: Add Frontend-Owned Help6529 Corpus

## PR

- GitHub PR: https://github.com/6529-Collections/6529seize-frontend/pull/2786
- Branch: `codex/6529-help-bot-spec`
- Target: `main`
- Related backend PR: https://github.com/6529-Collections/6529seize-backend/pull/1659

## Task Summary

Move Help6529 product knowledge ownership to the frontend and publish a validated public help corpus that the backend can fetch and cache.

## Frontend Change Summary

- Added `ops/help/help-index.json` with 140 curated glossary, route, workflow, and UI-affordance records.
- Added `scripts/sync-help-index.cjs` plus `help-index:sync` to validate corpus shape, source refs, internal routes, required records, and WordPress-migrated page exclusions.
- Added generated `public/help-index.json` and wired `prebuild` to regenerate and validate it.
- Added an exact proxy bypass for `/help-index.json` so staging can serve the public corpus without staging access auth.
- Updated Main Stage submission knowledge so "The Memes - Main Stage" is treated as a Waves submission context, not the `/the-memes` collection/mint browse route.
- Updated profile knowledge so the generic profile record points to `/network` and explains that exact profile pages must be opened from a real handle or wallet-like route target.
- Added public Memes calendar API responses for next, current, and specific Meme Card mint timing, including mint number, UTC start/end window, status, calendar path, and Meme Card path.
- Indexed Memes calendar timing aliases and facts so Help6529 can answer next/current/past drop timing and mint-window questions from the calendar API instead of generic collection docs.
- Kept WordPress-migrated and dated pages out of indexed knowledge.
- Audited the bot-handle rename; only `help6529` remains in the frontend branch, with no deprecated-handle literals retained.

## Frontend Impact Summary

- Runtime code changes: static public artifact, exact `/help-index.json` proxy access-control bypass, and public Memes calendar route handlers under `/api/meme-calendar`.
- User-visible UI changes: none.
- Public data shape: `/api/meme-calendar/next`, `/api/meme-calendar/current`, and `/api/meme-calendar/{id}` expose schedule and mint-window fields used by the backend bot.
- Build/deploy impact: frontend builds validate and publish `/help-index.json`.
