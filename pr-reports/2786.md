## Task Summary

Move `@6529help` product knowledge ownership to the frontend and publish a validated help corpus that the backend can consume quickly without live repo or code lookup.

## Frontend Change Summary

- Added `ops/help/help-index.json` with 149 curated glossary, workflow, route, and UI-affordance records covering Waves, TDH/Network, subscriptions, profiles, groups, delegation, Open Data, The Memes, Meme Lab, NextGen, Drop Forge, Rememes, Museum, OM, Education, Capital, Gradients, tools, and 6529bot/API-tool surfaces.
- Added `scripts/sync-help-index.cjs` plus `help-index:sync` to validate the corpus and generate `public/help-index.json`.
- Wired `prebuild` to sync the help index so missing source refs or bad internal links fail before deployment.
- Added an exact proxy bypass for `/help-index.json` so staging can serve the public corpus without staging access auth.
- Updated mobile navigation, pull-to-refresh, header, share, back-button, and loading affordance records to use `/waves` as their representative canonical surface instead of `/open-mobile` or `/`.
- Split generic route/not-found errors from restricted-access help so `/error` and `/restricted` are separate canonical records.
- Updated the help-index spec and agent guidance so future frontend changes maintain the corpus in the same PR.

## Frontend Impact Summary

- Runtime code changes: static public artifact plus proxy access-control bypass for the exact `/help-index.json` path.
- User-visible UI changes: none.
- Build/deploy impact: frontend builds now validate and publish `/help-index.json`.
