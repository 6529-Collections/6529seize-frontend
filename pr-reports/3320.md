# PR Report: Fix The Memes preview edition sizing

PR: https://github.com/6529-Collections/6529seize-frontend/pull/3320
Branch: `agent/fix-memes-preview-edition-sizes` -> `main`
Related PRs: None

## Summary

The Memes link previews now distinguish active mints from finalized editions, show honest live mint progress, and use a more compact chat layout.

## What Changed

- Use `recorded_in_tdh: false` as the explicit live-mint signal.
- Show a `Minting Live` badge and `Minted {current} / {maximum}` using API edition data and Manifold `totalMax`, with a current-count-only fallback.
- Use the backend-aligned `max(actual edition size, edition_size_floor)` calculation after TDH finalization or when lifecycle state is unknown.
- Keep mint-stat endpoints out of edition-size decisions.
- Reduce the medium-width stable frame height for The Memes previews without changing other collection cards.

## Frontend Impact

- Routes/views: Wave link previews for `/the-memes/{id}`.
- Components: First-party 6529 collection preview service, card, and chat stable frame.
- Generated API/client: Not affected.
- User-visible behavior: Live cards show mint progress rather than presenting the maximum as a completed edition size; finalized cards continue to show edition size.
