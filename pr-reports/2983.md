# PR Report: Unranked Meme TDH Stats

PR: https://github.com/6529-Collections/6529seize-frontend/pull/2983
Branch: `codex/recorded-in-tdh` -> `main`
Related PRs: https://github.com/6529-Collections/6529seize-backend/pull/1690

## Summary

Updates The Memes live stats UI to distinguish Memes that are explicitly not recorded in TDH from legacy or missing extended data.

## What Changed

- Added optional `recorded_in_tdh` and `ranked_collection_size` fields to the frontend Memes extended-data model.
- Shows rank pills as `Unranked` when `recorded_in_tdh === false`.
- Uses `ranked_collection_size` as the rank denominator when the backend provides it.
- Shows the TDH rate as `Pending` when a Meme is explicitly not recorded in TDH.
- Preserves current behavior when the new fields are absent or null.
- Added focused coverage for unranked rank pills, pending TDH rate, and ranked collection size denominators.

## Frontend Impact

- Routes/views: The Memes detail live stats panel.
- Components: `MemePageLive`, `MemePageLiveStats`.
- Generated API/client: Not affected.
- User-visible behavior: New or TDH-pending Memes show `Unranked` rank pills and `Pending` TDH rate instead of misleading numeric ranks/rates.

## Config / Env

None.

## Release Notes

The Memes detail page now renders TDH-pending cards as unranked and shows TDH rate as pending until backend TDH data records the card.
