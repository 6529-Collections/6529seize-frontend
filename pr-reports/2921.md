# PR Report: About contents dropdown navigation

PR: https://github.com/6529-Collections/6529seize-frontend/pull/2921
Branch: codex/about-contents-nav -> main
Related PRs: None

## Summary

Replaces the persistent About in-page side menu with a compact sticky Contents dropdown so About pages no longer show a second side navigation beside the global app sidebar.

## What Changed

- Added a shared About Contents dropdown that lists the requested About pages, highlights the current page, and navigates without changing existing `/about/{section}` paths.
- Preserved the existing Subscriptions iOS/country visibility rule and kept Cookie Policy reachable from the About navigation.
- Added link-backed dropdown items and documented the current About locale fallback debt for the progressive i18n migration.
- Removed the desktop side-menu column and mobile duplicate About menu from the main About route and About Tech subroutes.
- Updated Primary Address documentation that referenced the previous in-page About menu.

## Frontend Impact

- Routes/views: `/about/{section}`, `/about/tech/{reportSlug}`, and `/about/tech/wallet-authentication`.
- Components: About section layout and the new sticky Contents dropdown built on the existing compact menu primitive.
- Generated API/client: Not affected.
- User-visible behavior: About pages now have the global sidebar plus main content, with a small top-right page selector instead of a persistent in-page side menu.

## Config / Env

No config or environment changes.

## Release Notes

About section navigation is now available from a sticky Contents dropdown near the page header, reducing duplicated side navigation while preserving existing About page URLs.
