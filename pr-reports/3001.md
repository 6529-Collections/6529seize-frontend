# PR Report: Join 6529 Onboarding Hub

PR: https://github.com/6529-Collections/6529seize-frontend/pull/3001
Branch: codex/join-6529-journey -> main
Related PRs: None

## Summary

Adds `/join-6529`, a compact product-native onboarding hub that helps users pick
the right entry point into 6529: Waves, NFT browsing, The Memes subscriptions,
or creating and contributing.

## What Changed

- Reworked the route from a state-aware progress journey into a static
  onboarding hub with a compact hero, entry-path cards, a "how joining works"
  flow, focus sections, and beginner FAQ.
- Kept the page factual about wallet, profile, ETH, eligibility, subscriptions,
  Wave creation, and minting. The NFT copy avoids promising guaranteed mints.
- Removed route-local onboarding state orchestration, recent-drop checks, and
  localStorage progress flags from the Join page.
- Updated Join page i18n copy, navigation docs, Help6529 records, and the
  public help-index artifact.
- Preserved existing product routes and did not change auth, wallet, Wave,
  minting, or subscription logic.

## Frontend Impact

- Routes/views: Adds `/join-6529`.
- Components: Route-local client UI only.
- Generated API/client: Not affected.
- User-visible behavior: Users get a calm starting hub with direct links to
  Waves, The Memes, current mint, subscriptions information, and Wave creation.

## Backend Impact

Not affected.

## Config / Env

None.

## Release Notes

Adds a direct `/join-6529` onboarding hub that explains how to start with Waves,
NFTs, subscriptions, and creating on 6529 without implying that wallet
connection, ETH, or mint eligibility is required before browsing.
