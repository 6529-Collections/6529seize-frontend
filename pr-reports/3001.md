# PR Report: Join 6529 Journey Page

PR: https://github.com/6529-Collections/6529seize-frontend/pull/3001
Branch: codex/join-6529-journey -> main
Related PRs: None

## Summary

Adds `/join`, a state-aware onboarding and acquisition guide for users who
need to connect a wallet, authenticate, create a profile, enter public Waves,
and send a first public message.

## What Changed

- Replaced the `/join` placeholder route (from the navigation IA cleanup) with
  the full journey page and removed the placeholder i18n keys.
- Styled with the site palette (iron/primary) to match neighboring routes.
- Added a new App Router route at `/join` with progress, current-step CTA,
  and "Things to do next" action tiles.
- Wired the guide to live wallet/profile state, recent profile drops, and Wave
  participation signals so it can mark profile setup, profile image, Waves
  entry, and first public message progress.
- Kept profile image as an incomplete optional step when missing, without
  blocking Waves or messaging guidance.
- Updated docs and Help6529 knowledge records, including the web create-wave
  path `/waves?create=wave`.
- Pointed the fully complete state back to `/waves` for continued exploration.

## Frontend Impact

- Routes/views: Adds `/join`.
- Components: Adds route-local client UI only.
- Generated API/client: Not affected.
- User-visible behavior: Users get a guided, state-aware path into profile
  setup, Waves participation, subscriptions, delegations, help6529, and Wave
  creation.

## Backend Impact

Not affected.

None.
