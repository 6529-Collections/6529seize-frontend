# PR Report: Mobile Floating Dock Refinements

PR: https://github.com/6529-Collections/6529seize-frontend/pull/2865
Branch: codex/mobile-floating-dock-refine -> main
Related PRs: None
Mode: staging-deploy

## Summary

Refines the mobile floating dock after the initial rollout so list routes consistently use the floating treatment, detail routes get their composer space back, and active-state behavior remains stable across hidden/shown dock route transitions.

## What Changed

- Widened the expanded floating dock while keeping the compact dock size unchanged.
- Removed the pending-route highlight so the active indicator remains on the current route until it animates to the next active item.
- Switched waves, messages, and notifications list routes away from the fixed dock and onto the floating dock.
- Added notification bottom spacing and reversed notification scroll compaction to match the bottom-starting feed.
- Hid the dock on wave and message detail routes while keeping the existing slide-away hidden state.
- Scoped element-scroll compaction to marked mobile stream scroll containers and batched all scrolled targets per animation frame.
- Cleared canceled scroll animation frame handles so compaction resumes cleanly after dock hide/show transitions.
- Replaced the shared-layout active pill animation with a local active indicator to avoid stale geometry when navigating from dock-hidden detail pages back to dock-visible pages.
- Lowered and slightly enlarged the floating dock, tightened notification bottom spacing, preloaded/crossfaded the 6529 logo active colors, and matched the wave/message detail composer safe-area strip to the composer surface.
- Added a small safe-area floor to wave/message detail composers and forced the composer placeholder to stay on one truncating line when action buttons are expanded.

## Frontend Impact

- Routes/views: `/waves`, `/messages`, and `/notifications` use the floating dock; `/waves/[id]` and `/messages/[id]` hide it.
- Components: Updates `BottomNavigation`, `NavItem`, `AppLayout`, and notifications reverse-list spacing.
- Generated API/client: Not affected.
- User-visible behavior: Mobile dock sizing/position, active-state rendering, route visibility, detail composer spacing, and notification bottom spacing are updated.

## Config / Env

None.

## Release Notes

Improves the mobile floating dock rollout by using it on stream list pages, removing the transient pending highlight, hiding it on wave/message detail pages, and stabilizing composer/dock behavior found during staging review.
