# PR Report: Capacitor Page Share Header Action

PR: https://github.com/6529-Collections/6529seize-frontend/pull/3002
Branch: codex/capacitor-page-share -> main
Related PRs: None

## Summary
Adds a native-app page share control to the app header for shareable pages outside Waves, Messages, and Notifications, while keeping wave-specific sharing separate.

## What Changed
- Added a Capacitor-only `Share page` header action that opens the native share sheet with the current path, query string, and hash preserved on the public web origin.
- Replaced the native app header's single-item wave-share overflow menu with a direct icon-only share/copy button; when other wave actions exist, the ellipsis menu remains and keeps the `Share wave` label.
- Kept the direct native wave share icon stable after dismissing the share sheet and added a pressed/busy cue for both native page share and direct wave share.
- Removed the duplicate `Create DM` overflow item because Messages already has a direct paper-plane action.
- Updated header tests, message keys, navigation docs, and help index records.

## Frontend Impact
- Routes/views: native app header surfaces outside `/waves*`, `/messages*`, and `/notifications*`.
- Components: app header overflow handling and new `HeaderPageShareButton`.
- Generated API/client: not affected.
- User-visible behavior: supported native app pages now show a share icon left of Search; wave and message surfaces keep their own controls without duplicate overflow items.

## Config / Env
No new config or environment variables. The native page share URL uses the existing public `BASE_ENDPOINT` runtime value.

## Release Notes
Native app users can share supported pages directly from the top header, with exact URL state preserved. Wave routes continue using wave-specific sharing, and the Messages header no longer shows a redundant one-item `Create DM` overflow menu.
