# PR Report: Active Wave Reaction Read

PR: https://github.com/6529-Collections/6529seize-frontend/pull/2787
Branch: codex/wave-reaction-active-read -> main
Related PRs: None
Mode: staging-deploy

## Summary
Realtime reaction updates in an open wave now reuse the same active-wave read path as new drop updates, so visible incoming reactions are treated as read by the frontend and do not leave the reaction notification pending for push delivery.

## What Changed
- Marked the active visible wave as read after a `DROP_REACTION_UPDATE` websocket event is reconciled for a loaded drop.
- Reused the existing delivered-notification cleanup and wave notification read API call.
- Added a focused regression test for active-wave reaction read behavior.

## Backend Impact
Not affected.

## Frontend Impact
- Routes/views: Wave chat and wave drop realtime views.
- Components: `useWaveRealtimeUpdater` active-wave read behavior.
- Generated API/client: Not affected.
- User-visible behavior: Users actively viewing a wave should no longer receive push notifications for reaction updates they see arrive in realtime.

## Config / Env
No new or changed config.

## Release Notes
Active waves now treat visible realtime reaction updates as read, matching the existing behavior for new wave messages.
