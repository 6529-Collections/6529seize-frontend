Registering delegations allows you to mint any allowlist spots from your most convenient wallet. The optimal method for setting delegations in TAP is described below.

Assuming three sample addresses:

  • Address A (vault) : 0xAcf42B85eCb77d9332584119FD78a3DE9953c2a0

  • Address B (transaction wallet) : 0x3558C942EeA9e9Bb9b1a6A02d272756EDD3A1Fe4

  • Address C (minting wallet) : 0xD0f03CB42Fe821c0654846E474c19484865c5B4a


  1. Connect to Seize.io with Address A (vault):

    • Set Address B (transaction) as the Subdelegate of Address A.

    • Go to the Delegation Center section and click the Register Delegation Manager (Sub-Delegation) button.

    • Choose 'Any Collection' and enter the wallet Address B.


    • Click on 'Submit' and execute the transaction.

    Once completed, Address A will not need to connect to any service again, and further delegations can be made on its behalf by Address B.

  2. Connect to Seize.io with Address B (transaction):

    1. Delegate Case 2 (minting/allowlist) to Address C (minting).

      • Go to the Delegation Center section and click the Register Delegation button.

      • Choose 'Any Collection,' enter wallet Address C, and select Use Case #2 Minting / Allowlist.

      • Choose an expiry date (e.g., Never), specify if the delegation applies to all tokens owned by Address B or a specific token ID (it is recommended not to select a specific token ID).

      • Click on 'Submit' and execute the transaction.


    2. Delegate Case 2 (minting/allowlist) on behalf of Address A (vault) to Address C (minting).

      • Go to the Delegation Center section, click the View/Manage button on 'Any Collection,' and locate the incoming Delegation Manager under the Delegation Managers (Sub-delegations) area.


      • Select the Address of wallet A and click the Register Delegation button.


      • Complete the Register Delegation Using Delegation Management (Sub-delegation) Rights form:

        • Choose 'Any Collection' and enter wallet Address C.

        • Select Use Case #2 Minting / Allowlist.

        • Choose an expiry date (e.g., Never) and specify if the delegation applies to all tokens owned by Address A or a specific token ID (it is recommended not to select a specific token ID).

        • Click on 'Submit' and execute the transaction.




      This will assign all allowlist spots for Addresses A, B, and C to Address C, which can mint on behalf of all three. Address B can continue to manage delegations for Address A.

    3. Connect to Seize.io with Address C (minting):

      • [To confirm] Go to the Delegation Center section, click on the View/Manage button on 'Any Collection,' and view the two delegation addresses in the Delegations area: one for wallet A and one for wallet B.
